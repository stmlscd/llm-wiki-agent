# 20260412 회장기 랭킹 포인트 산출 및 DB 업데이트

## 배경
- 52회 회장기 대회 종료 후, VPS 가상서버(115.68.177.35)에 모든 경기 결과가 기록되어 있음
- 랭킹 메뉴에 종별선수권 등 기존 대회 점수는 반영되어 있으나, 회장기(tournament_id=1) 점수는 0건으로 비어있었음
- 사용자가 `회장기-랭킹포인트.xlsx` 파일을 제공하여 점수 기준 확인

## 회장기 랭킹 포인트 기준 (R1 등급)

| 배치 | 6학년 | 5학년 | 4학년 |
|------|-------|-------|-------|
| 우승 | 60 | 50 | 40 |
| 준우승 | 50 | 40 | 30 |
| 4강 | 40 | 30 | 20 |
| 8강 | 30 | 20 | 15 |
| 16강 | 20 | 15 | 10 |
| 32강 | 15 | 10 | 5 |
| 64강 | 10 | 5 | 0 |

### 특수 규칙
- **참가점수**: 대회 참가 선수 전원에게 10점 부여
- **예선 3위까지**: 본선 진출하지 못했더라도 64강 점수 부여
- **예선 4위 이하**: 참가점수(10점)만 부여
- **불참/기권**: 참가점수 0, 성적점수 0

## 작업 과정

### 1단계: 데이터 수집
- VPS MySQL(kettf_tournament)에 SSH(paramiko)로 접속 (root 계정 — kettf_admin은 접속 거부됨)
- `tournament_matches` 테이블에서 4,5,6학년 남녀 6개 카테고리 전 라운드 결과 추출
- `tiebreak_results` 테이블에서 예선 조별 순위 추출
- `draw_results` 테이블에서 본선 진출자 매핑
- `players` 테이블에서 전체 참가 선수 목록 추출

### 2단계: 배치(Placement) 계산
- 결승 winner → 우승, loser → 준우승
- 준결승 loser → 4강
- 8강 loser → 8강
- 16강 loser → 16강
- 32강 loser → 32강
- 예선에서 본선 미진출 + rank ≤ 3 → 64강
- 나머지 → 참가(10점만)

### 3단계: DB 업데이트
- `rk_players`: 기존 342명 + 신규 39명 = 381명
- `rk_scores`: 회장기(tournament_id=1)에 291건 INSERT
- `rk_rankings`: 전체 성별/학년별 dense ranking 재계산 (382건)

### 4단계: 검증
- VPS API(`/api/ranking/남/6`) 호출로 정상 반영 확인
- 강도현(남6): 회장기 60점(준우승50+참가10), 총합 270점 = 1위

### 5단계: 엑셀 파일 생성
- `52회_회장기_랭킹_남자.xlsx` (201명) — 4학년/5학년/6학년/통합 시트
- `52회_회장기_랭킹_여자.xlsx` (181명) — 동일 구조
- 회장기, 종별선수권 컬럼에 노란색 하이라이트 적용

## 기술적 이슈
- **kettf_admin 계정 접속 거부**: VPS MySQL에 `kettf_admin` 계정으로 접속 시 Access denied 발생. root 계정 사용으로 우회.
- **pymysql 직접 접속 불가**: MySQL 3306 포트가 외부에 열려있지 않아 paramiko SSH 경유 필요.
- **터미널 인코딩**: Windows bash에서 한글 출력 깨짐 — JSON 파일로 중간 저장하여 해결.
