# Codex 코드 리뷰 — 호환 라우터 (2026-04-17)

**대상**: 작업 트리 diff (working tree diff)

호환 라우터(compat router)에서 보안 취약점 1건과 대진표 크기 오류 1건이 발견되었다.
이 두 문제가 해결되기 전까지는 해당 패치를 올바른 것으로 간주할 수 없다.

---

## [P1] 쓰기 API에 인증 검사 누락

**파일**: `_new_routes.txt:575-579`

POST 핸들러들이 인증된 관리자(admin)를 요구하지 않는다. 기존 Flask 라우트에서는 인증이 필요했던 엔드포인트들이다.
메인 페이지가 공개 접근 가능한 배포 환경에서, 인증 없는 클라이언트가 다음 작업을 수행할 수 있다:

- `/match/...` — 경기 결과 입력
- `/api/result` — 결과 재계산
- `/rankings/.../calculate` — 랭킹 계산
- 단체전 엔드포인트 — 팀 오더/결과 수정

**영향**: 데이터 무결성 및 보안에 직접적인 위험. 기존 Flask 대비 보안 퇴보.

**조치**: 모든 변경(mutating) API에 관리자 인증 검사를 반드시 복원해야 한다.

---

## [P1] 16조 초과 시 64강 대진표 미지원

**파일**: `_new_routes.txt:336-343`

`api_draw_info()`가 개인전 대진표를 `group_count` 기준으로 16강 또는 32강으로만 하드코딩한다.
그러나 대회는 각 조에서 상위 2명이 본선에 진출하므로, 16조를 초과하면 32명 이상의 진출자가 발생한다.

이 경우 반환되는 `bracket_size`, BYE 계산, 슬롯 목록이 모두 틀리게 된다.
라우터 자체에 64강 매핑과 라운드 순서가 정의되어 있음에도 사용되지 않는다.

**조치**: 조 수 > 16일 때 64강 대진표를 지원하도록 수정해야 한다.

---

## [P2] 경기 저장 시 사유(reason) 미보존

**파일**: `_new_routes.txt:610-617`

52nd 경기 모달에서 기권(`기권`)/실격(`실격`) 등의 사유를 입력할 수 있고,
`bracket_data()`에서 `m.reason`을 읽지만, `match_save()`는 요청에서 `reason`을 가져오거나 저장하지 않는다.

기권/실격으로 입력된 경기를 다시 열거나 대진표를 렌더링하면 해당 상태가 사라지고
일반 점수 입력으로 취급된다.

**조치**: `match_save()`에서 `reason` 필드를 요청에서 읽어 DB에 저장하도록 수정해야 한다.

## Adversarial 리뷰 (같은 날)

### [HIGH] 서브모듈 dirty 상태 — 감사/재현 불가

**위치**: `llm-wiki-agent`

`llm-wiki-agent` 서브모듈이 새 커밋으로 갱신되지 않고 `-dirty` 상태로만 기록되어 있다.
실제 변경된 코드가 부모 diff에 캡처되지 않고, 다른 사람이 checkout해도 동일 상태를 재현할 수 없다.
서브모듈 내부의 어떤 변경이든 안정적인 리비전 없이 출하될 수 있으며, 롤백/복구가 추측에 의존하게 된다.

**조치**: 서브모듈 변경사항을 커밋하고 gitlink를 실제 리비전으로 업데이트하거나, 출하 전에 서브모듈을 clean/reset해야 한다.

### [MEDIUM] 외부 플러그인이 레포 설정에 강제 활성화

**위치**: `.claude/settings.json:29-30`

`enabledPlugins`에 `codex@openai-codex`가 추가되어, 이 레포를 여는 모든 협업자에게 자동으로 외부 플러그인이 활성화된다.
의도하지 않은 데이터 노출, 환경별 다른 동작, 롤백 어려움 등의 문제가 있다.

**조치**: 레포 설정에서 플러그인을 기본 활성화하지 말고, 사용자 로컬 설정으로 이동하거나 명시적 opt-in 메커니즘 추가 필요.
