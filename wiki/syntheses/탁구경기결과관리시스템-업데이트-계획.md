### Phase 1 — 대학연맹 시스템 (새로 시작, FastAPI)
**목적: 깨끗한 구조로 처음부터 → [[UniversityLeague]]**

**기술 스택 (결정 완료)**
```
FastAPI + Alembic + Pydantic + Alpine.js + Jinja2
```
→ [[TechStackMigration]] 참고

**핵심 설계 원칙 — 기능 단위 격리**

```
현재 (모노리스)               →   다음 (기능별 분리)
──────────────────────────────────────────────────
main.js (3000줄)              →   league.js   (리그 입력)
                                  bracket.js  (대진표)
                                  team.js     (단체전)
                                  awards.js   (시상)
```

Alpine.js를 쓰면 각 탭이 독립 `x-data` 컴포넌트가 되어 하나를 건드려도 다른 탭은 절대 영향 없음.

### Phase 3 — 새 기능 추가 규칙 (재발 방지)

새 기능이 기존 기능을 깨지 않으려면 코드 레벨 분리가 필수.

```
새 기능 추가 규칙
─────────────────────────────────────────────
1. 새 기능 = 새 파일 또는 새 x-data 블록
2. 기존 API endpoint는 절대 수정하지 않고 새 endpoint 추가
3. 기능 추가 후 smoke_test.md 전체 항목 확인 필수
4. 배포 전 tar.gz 스냅샷 먼저
```

**대학연맹에서 처음부터 구현할 기능들**
- 단체전 경기 방식 프리셋 (4단1복 / 5단식 / 5단2복) 사전 정의 선택 구조
- 랭킹 UI (학년별·성별 순위표 프론트엔드)
- 코치/팀 로그인 포털
- 선수 관리 CRUD UI (SSH 없이 관리자 페이지에서 추가/수정)
- Pydantic Enum으로 경기 방식 오입력 차단
